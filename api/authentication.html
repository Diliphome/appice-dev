Authentication

Every Appice API request authenticates with two headers: a Bearer API key and your workspace ID. The combination scopes the request to one workspace in one region.

Required headers

Authorization: Bearer <YOUR_API_KEY>
X-Appice-Workspace: <YOUR_WORKSPACE_ID>
Content-Type: application/json
Server-side only — Server keys must never ship to a browser, mobile app, or any environment outside your trusted backend. For client telemetry, use the per-platform SDK with its scoped client key.

Where to get keys

  1. Sign in to panel.appice.io.
  2. Open Settings → API Keys.
  3. Click Generate server key. Copy the key immediately — it's only shown once.
  4. Your workspace ID is shown in Settings → Workspace.

Don't have a workspace yet? Request sandbox access — turnaround is typically same-day.

Regional endpoints

Each workspace is provisioned in exactly one region. Data residency is enforced server-side — calling the wrong region's endpoint with valid credentials returns 401 Unauthorized.

Global: https://api.appice.io/v1
India:  https://api.in.appice.io/v1
GCC:    https://api.gcc.appice.io/v1
EU:     https://api.eu.appice.io/v1

Examples

cURL

curl -X POST https://api.in.appice.io/v1/events \
  -H "Authorization: Bearer $APPICE_API_KEY" \
  -H "X-Appice-Workspace: $APPICE_WORKSPACE" \
  -H "Content-Type: application/json" \
  -d '{
    "events": [{
      "userId": "user_4711",
      "event": "Loan Disbursed",
      "timestamp": "2026-05-06T10:33:11Z",
      "properties": { "amount": 250000, "currency": "INR" }
    }]
  }'

Node Express (server-side)

import fetch from 'node-fetch';

const res = await fetch('https://api.in.appice.io/v1/events', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.APPICE_API_KEY}`,
    'X-Appice-Workspace': process.env.APPICE_WORKSPACE,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    events: [{
      userId: 'user_4711',
      event: 'Loan Disbursed',
      timestamp: new Date().toISOString(),
      properties: { amount: 250000, currency: 'INR' }
    }]
  })
});

Key rotation

Server keys can be rotated without downtime. The recommended pattern:

  1. Generate a new key in panel.appice.io — both keys are now valid.
  2. Roll the new key out to your backend (env var update + restart).
  3. Verify traffic on the new key in Settings → API Keys → Activity.
  4. Revoke the old key once activity drops to zero.

Keys expire after 365 days of inactivity. Active keys never expire automatically.

Scopes (planned, Q3 2026)

Today, server keys have full workspace access. Scoped keys (limited to events:write, inform:trigger, webhooks:read, etc.) are on the roadmap — watch the changelog.

Troubleshooting 401s

  • Region mismatch: check the host you're calling matches the workspace's provisioned region.
  • Header typo: X-Appice-Workspace not X-Appice-WorkspaceId.
  • Key revoked: check Settings → API Keys for status.
  • Clock skew: requests with timestamps more than 5 minutes off server time are rejected. Sync your server clock via NTP.
